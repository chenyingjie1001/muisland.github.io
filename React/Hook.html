    # HOOK

    # 针对的问题

    在hooks之前实现组件的逻辑复用都是用HOC，render Props, 他们的本质是将通用逻辑提升到父组件中，很容易产生很多包装组件，带来地狱嵌套

    # 优势

    • 逻辑清晰，容易复用代码，不需要层层嵌套
    • 使得函数组件也拥有状态，相比class组件，代码量减少，也不需要担心this问题

    # 工作原理

    react创建一个函数组件时，会创建一个对象与之共存，只要函数组件一直存在于dom中，对象也一直存在。在对象中设置了一个hooks数组，每次调用一个hook,就会向数组添加一个hook,第二次渲染时，惠安相同顺序调用hook,如果hook有值，则返回该值，不会进行初始化。

    var memorizedState = []
var cursor = 0
function useState(initialState){
   const currentCursor = cursor
   memorizedState[currentCursor] = memorizedState[currentCursor] || initialState
   function setState(state){
     memorizedState[currentCursor] = state
   }
   return [memorizedState[cursor++], setState]
}
function useEffect(callback, depArray) {
   const hasNoDeps = !depArray
   const deps = memorizedState[cursor]
   const hasChangeDeps = deps ? !deps.every((d, index) => d === depArray[index]) : true
   if(hasNoDeps || hasChangeDeps){
      callback()
      memorizedState[cursor] = depArray
   }
   cursor++
}

    # useEffect

    • 依赖项数组为空，每次渲染都更新
    • 数据有值，当依赖项有值才更新

    # useCallback

    处理一些数据更新引起的其他组件不必要的渲染

    const memorizedCallback = useCallback((fn, deps) => {
   dosomething(a, b)
}, [a, b])
    • 只有依赖项改变时，内联函数中的值才会得到最新的值，否则都是之前的值
    • 当依赖项为空, 返回的回调函数一直不会改变

    # useMemo

    优化避免在每次渲染时都进行高开销的计算

    const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);
    • 若没有提供依赖项，useMemo每次渲染都会计算新的值
    • 若提供依赖项，某个依赖项改变才会渲染

    # useRef

    ref可以放任何东西，每次渲染同一个引用，createRef返回不同的引用

    # useImperativeHandle

    避免暴露过多的子属性给父组件
